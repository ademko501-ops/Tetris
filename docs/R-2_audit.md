## ЗАМЕЧАНИЯ

### 1. [приоритет: низкий] `src/style.css`:36-37, 45-46 — магическое число `30px` встречается четыре раза

В CSS сетка поля задаётся через `repeat(12, 30px)` и `repeat(24, 30px)`, и тут же класс `.cell` прописывает `width: 30px; height: 30px`. Размер клетки `30px` — это ключевая «единица масштаба» всей игры. Он повторяется 4 раза без имени. Если в следующих этапах захочется поменять масштаб (например, сделать 32 px для красоты), придётся искать и менять все четыре вхождения.

Что предлагаю: добавить в начало `style.css` CSS-переменную `--cell-size: 30px` и использовать её вместо голого значения в трёх местах: в `grid-template-columns`, `grid-template-rows`, `.cell width`, `.cell height`. Это займёт одну строку добавления и четыре замены.

### 2. [приоритет: низкий] `src/style.css`:38 — `rgba(21, 255, 0, 0.4)` — дублирование цвета палитры в другом формате

Цвет свечения поля записан как `rgba(21, 255, 0, 0.4)` — это тот же `#15ff00`, но в числовом формате RGB. Палитра проекта ведётся в HEX-обозначениях (`#15ff00`, `#ffb000`, `#0a0a0a`). Нарушения стиля формально нет — цвет правильный — но запись `21, 255, 0` не очевидно связана с `#15ff00` при беглом чтении кода.

Что предлагаю: когда будет введена CSS-переменная для основного цвета, рядом можно завести `--color-green-glow: rgba(21, 255, 0, 0.4)`. Пока не критично — файл маленький и это единственное такое вхождение.

### 3. [приоритет: средний] `src/style.css`:36-37 vs `src/game.js`:11-12 — размеры поля (12×24) живут в двух местах независимо

В `game.js` строки 11-12 объявлены константы `ROWS = 24` и `COLS = 12`. В CSS строки 36-37 те же числа вбиты прямо в `repeat(12, 30px)` и `repeat(24, 30px)`. Оба места не связаны — это два независимых источника одной истины. Если на следующих этапах поле поменяет размер, нужно будет помнить поменять и в JS, и в CSS.

Что предлагаю: для CSS добавить переменные `--cols: 12` и `--rows: 24` (или использовать уже упомянутую `--cell-size` вместе с вычислениями). На текущем этапе несинхронность не ломает ничего, но лучше зафиксировать её как задачу на следующий этап.
