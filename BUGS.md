# BUGS — Список багов

## #1. Стрелка ↓ скроллит страницу параллельно с ускорением падения (R-5)
- **Симптом:** при удержании ↓ фигура падает быстрее (как и задумано),
  но браузер одновременно прокручивает страницу вниз, и контент уезжает
  из видимой области.
- **Причина:** обработчик `keydown` в `src/game.js` не вызывал
  `event.preventDefault()`, поэтому отрабатывало действие браузера
  по умолчанию (прокрутка страницы стрелками).
- **Найден:** при ручной проверке архитектором, уже после прогона R-5
  через vault-reviewer. Ревьюер не поймал — это UX-проблема, а не
  проблема качества кода.
- **Исправлен в:** `0d92bdf` — добавлен `event.preventDefault()` для
  `ArrowDown`, `ArrowLeft`, `ArrowRight` в начале keydown-обработчика
  (`ArrowUp` и `Space` оставлены без preventDefault — зарезервированы
  под R-7 / R-8).
- **Сопутствующий рефакторинг:** `d11a4e4` — список этих клавиш вынесен
  в константу `CONTROLLED_KEYS`, чтобы при добавлении новых клавиш правка
  делалась в одном месте (закрыт средний приоритет по аудиту fix-коммита).

## #2. Зажатая ↓ + потеря фокуса окна → таймер навсегда на ускоренной скорости (известное ограничение)

- **Тип:** известное ограничение, не баг в обычном сценарии.
- **Симптом:** игрок держит ↓ (фигура падает ускоренно), окно теряет
  фокус (Alt+Tab, клик в другое приложение). Игрок физически отпускает
  клавишу, но событие `keyup` приходит уже в чужое окно — наша страница
  его не видит. При возврате фокуса фигура продолжает падать ускоренно
  до самого верха стека, даже без удерживаемой ↓.
- **Причина:** обработчик `keyup` слушает только пока окно в фокусе.
  Сейчас нет страховки на `window.blur` / `visibilitychange`, которая
  принудительно вернула бы обычную скорость падения.
- **Найден:** vault-reviewer на ревью R-6 (cfcb7a8), специальная
  проверка регрессий после удаления `isFalling`.
- **Статус:** **не исправлен**, отложен. В обычной игре проявляется
  редко, без падений и потери данных. Естественное место для лечения —
  R-11 (Game Over / пауза), где появится `window.addEventListener('blur', ...)`
  для глобальных событий жизненного цикла окна.
- **Воркараунд игроку:** если фигура «зависла» в ускоренном режиме,
  достаточно один раз нажать и отпустить ↓ при активном окне — таймер
  переключится обратно.
